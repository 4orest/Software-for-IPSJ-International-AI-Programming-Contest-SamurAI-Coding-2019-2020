SamurAI Dig Here ゲームマネージャ

情報処理学会プログラミングコンテスト委員会
2019/10/01

SamurAI Dig Here ゲームマネージャは, ふたつのプレイヤエージェントプログラム同士で SamurAI Dig Here ゲームを競わせ, そのゲームのログを生成するツールである.

機能

SamurAI Dig Here ゲームマネージャは以下の機能を持つ.

ゲームの実施:
与えられたゲーム競技場において, ふたつのプレイヤエージェントプログラムを競わせ, ゲームのログを出力する.
通信ログの生成:
ゲーム実施中にマネージャからプレイヤプログラムに送ったデータのコピーを外部ファイルに格納できる. これはプレイヤプログラムのデバッグに有用だろう.
プレイログの確認:
与えられたプレイログの整合性をチェックできる.

使用法

コマンドラインは以下のフォーマットである. 

./manager [options] <log> [<player1> <player2>]

競技場

引数 log は競技場の記述と (空かも知れない) その上でのゲームのログを格納するログファイルへのパスである. このファイルは通常 SamurAI Dig Here ウェブページ を用いて作成するもので, ファイル名拡張子 ".dighere" を持つ.

プレイヤプログラム

引数 <player1> と <player2> を与えなければ, マネージャはログファイルに記された内容の整合性をチェックする.

引数 <player1> と <player2> を与えれば, これらはプレイヤプログラムへのパスと解釈される. マネージャは引数 <log> に与えられた競技場を用いて, これらふたつのプレイヤにゲームを競わせる. 詳細は後述する.

オプション

以下のオプションがある.

-S:
ゲームの各ステップのサマリを標準エラー出力に書き出す. サマリは以下の行からなる. 
Step: 8
Golds known: 18@(6,6) 14@(6,8)
Agent 0@(6,7) dig  (6,8)
Agent 1@(5,6) move (5,7)X
Agent 2@(4,8) move (3,8)
Agent 3@(6,6) move (5,7)X
Scores are 14:0
  • ゲームステップ番号.
  • 公開されている埋蔵金の量と位置.
  • 4 エージェントの位置とプレイプランを各1行に. 移動, 穴掘り, 埋め戻しをするセルの位置も表示する. 実際にはプレイプランが他のエージェントのプランと衝突する場合は, X を付してそのことを示す.
  • 当該ステップまでの両チームの得点.
右図に例を示す. この例では片方のチームの侍 (agent 1) と犬 (agent 3) が同じセルに動こうとしたので衝突が生じ, 両者は元の場所から動けない. 公開されている (6,8) にある埋蔵金は相手の侍 (agent 0) が掘り出した.
-D <path>
マネージャからプレイヤプログラムに送るデータのコピーを指定ファイルに書き出す. エージェントは4体あるので, 出力先は4ファイルになる. 各ファイルのパスは指定した <path> にエージェントの ID を付加したものになる. ID は以下である.
0:Player 1 の侍
1:Player 2 の侍
2:Player 1 の犬
3:Player 2 の犬

プレイヤプロセス

プレイヤプログラムはマネージャの子プロセスとして動作する. 両チームそれぞれ 2 プロセス, ひとつは侍, もうひとつは犬を制御するプロセスが同じプログラムで動作する. どちらのエージェントの担当かはマネージャから送られるデータでわかる.

各プロセスは, ゲームステップごとに標準入力に送られるゲーム状態のデータを読み込み, そのステップのプレイプランを標準出力に書き出す. 詳細は SamurAI Dig Here ゲームルール を参照されたい.
